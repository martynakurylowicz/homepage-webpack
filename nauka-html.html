HTML składa się ze znaczników / Tagów / Elementów

<znacznik atrybut="wartość">treść</znacznik>

Zaczynamy pisanie od !+tab

Poznane znaczniki:

<main> - główna część dokumentu

<section> - sekcje dokumentu np. tak jak sekcja sportowa w gazecie

<article> - samodzielne treści np. tak jak pojedyńczy artykuł w sekcji sportowej

<header> - nagłówek dokumentu / sekcji / artykułu

<footer> - stopka dokumentu / sekcji / artykułu

<h1-h6> - tekstowe nagłówki tytułowe - dokumentu / sekcji

<p> - paragraf / akapit

<br> - nowa linia - znacznik pojedynczy

<b><i> - wyróżniki

<strong><em> - wyróżniki - m.in. dla przeczytania jako ważne lub ironia

<a href = ""target"" target="blank/window/self">opis łącza<a/> - linki

<element id="moj_identyfikator"> - wewnętrzne łącza

<a href="#moj_identyfikator">opis łącza<a/> - wewnętrzne łącza - przejście np. do sekcji z opisanym id

<nav> - główna nawigacja - poza main-em

<img src="" alt=""> - obrazek

<znacznik style="color:red"; "background:blue">treść </znacznik> - zmiana stylu wybranego znacznika

<znacznik class="name">treść </znacznik> - selektor stylu - dla CSS / "name" warto wpisać z użyciem kebab-case

<znacznik id="name">treść </znacznik> - selektor stylu - dla CSS / "name" warto wpisać z użyciem kebab-case / id jedno na stronie - nie używać przy wielu elementach

<style> - styl zamiast .css /przy danym elemencie

<script> - skrypt kodu zamiast .js / wrzuca się przez zamknięciem znacznika body - na końcu

<script type="module"> - skrypt kodu z gotowego modułu

<div> - pudełko / opakowanie

<ul> - nieuporządkowana lista - automatycznie kropki

<ol> - uporządkowana lista - liczby

<li> - element listy

<ol> - lista zagnieżdżona w innej liście / podpunkty

<dl> - lista z definicją / dt - tytuł, dd - treść

<aside> - treść boczna / jak article

<button> - przycisk

Tabele:

<table> - tabela

<tr> - element tabeli - wiersz (row)

<td> - element tabeli - komórka tabeli

<thead> - element tabeli - nagłówek

<th> - element tabeli - komórka w nagłówku / kolumny

<tbody> - element tabeli - ciało tabeli

<caption> - opis tabeli

Formularze:

<form> - formularz - opakowanie

<input> - do wpisywania / opakować w div lub paragraf

<input name="nazwaPola"> - każdy input musi mieć swoją nazwę/zmienną

<input id="nazwaPola" name="nazwaPola"> - idenyfikacja pola

<label> - opis pola

<label for="nazwaPola"> - przypisujemy do inputa po id

<placeholder="Proszę wpisać imię"> - dodany w inpucie

<fieldset> - grupa pól np. do adresu

<legend> - opis grupy pól

<input type="radio"> - wybór pomiędzy opcjami - domyślnie kropki
np.<label><input name="iceCream" type="radio" value="chocolate">Chocolate</label>

<input type="ckeckbox"> - wybór pomiędzy opcjami / można zaznaczyć kilka - domyślnie kwadraty

<input checked type="radio"> - domyślnie zaznaczony

<input type="number/tel/email"> - typy do precyzowania danych

<input type="range" min="0" max="100" step="5"> - typ do wybierania zakresu - suwak

<input type="password"> - typ do hasła - zasłania wpisane wartości

<input type="file"> - do załączenia pliku

<select> - zrobienie listy do wyboru - element listy <option> / wybór z rozwijanego paska

<textarea> - do opisywania - do wprowadzania dłuższej treści jak wiadomość

<input type="submit" value="send"> - wysyłanie formularza

action="url://z-ktorym-gadamy" - adres do wysłania - żądanie typu GET

method="post" + action - żądanie typu POST

enctype="application/x-www-form-irlencoded" - wartość domyślna form
enctype="multipart/form-data" - używamy, kiedy wysyłamy pliki

required - po dodaniu do elementu formularza wyświetli informację, że musi zostać wypełniony przed wysłaniem

pattern="[a-zA-Z0-9]+" - wyrażenia regularne
